# Optimizer Comparison: PSO vs Hybrid DE vs DE-MCMC

This directory contains an experiment (`compare_optimizers.py`) comparing three calibration workflows built into `pyair2stream`:
1. The native Particle Swarm Optimizer (PSO).
2. The modern, hybrid Differential Evolution + L-BFGS-B (DE) method.
3. The DE-MCMC method, which adds a Markov Chain Monte Carlo (MCMC) uncertainty analysis stage.

The experiment was run against the dataset generated in `examples/synthetic_example/`.

## Why a Hybrid Optimizer?

Particle Swarm Optimization works well for roughly narrowing down bounds globally but tends to converge extremely slowly on exact local optimums (or fails to entirely, getting caught in local minima).

For an 8-parameter problem, **Differential Evolution (DE)** explores the parameter space significantly better and is less prone to premature convergence. Once DE finds the correct underlying basin of attraction, switching to **L-BFGS-B** (which uses numerical gradients) allows us to mathematically "snap" to the true local optimum instantly, skipping thousands of fine-tuning evaluations.

## Why DE-MCMC?

While DE + L-BFGS-B reliably finds a global optimum, it only provides a single point estimate for parameters. Hydrological models like `pyair2stream` often suffer from "equifinality", where multiple different parameter sets produce nearly identical objective scores.

By passing the optimal DE solution into an MCMC sampler (using `emcee`), we can map out the full posterior distribution of the parameters. This allows us to extract **Predictive Uncertainty Envelopes** (e.g., the 5th and 95th percentiles of simulated temperatures), providing critical confidence intervals for climate change projections or regulatory reporting.

## Results

Executing the configuration (`50 particles/popsize`, `100 runs/maxiter`) yields:

### Particle Swarm Optimization (PSO)
* **Execution Time:** ~5.67 seconds (With Numba JIT)
* **Accuracy (NSE):** 0.953845
* **Parameters Found:** `[5.044, 0.74865, 0.77513, 0.314, 0.94073, 4.3796, 0.53936, 0.67431]`

### Differential Evolution + L-BFGS-B (DE)
* **Execution Time:** ~2.82 seconds (With Numba JIT)
* **Accuracy (NSE):** **0.970449** (Substantial improvement)
* **Parameters Found:** `[0.1, 0.00622, 0.0087, 1.0, 1.0, 0.76149, 0.51917, 0.09169]`

### DE-MCMC (Uncertainty Analysis)
* **Execution Time:** ~20.48 seconds (For `32 walkers` and `2000 steps` with 30% burn-in discarded).
* **Accuracy:** Matches DE (as it seeds from the DE optimum).
* **Benefit:** Produces parameter chains and a remarkably tight, converged predictive 90% envelope.

## Conclusion

While PSO is rapid for prototyping, the **Hybrid Differential Evolution + L-BFGS-B** method provides objectively superior mathematical convergence to the true global optimum.

Furthermore, in scientific research scenarios, using the **DE-MCMC** workflow with prolonged convergence steps and strict burn-in discard policies to calculate explicit, narrow uncertainty envelopes is highly recommended over relying on single point estimates.
